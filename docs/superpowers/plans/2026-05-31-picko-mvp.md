# Picko MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first testable Picko MVP slice: shared organizing logic for review decisions, similar groups, keep-count selection, undo, and the pre-delete basket, then use it from iOS/macOS UI targets in later phases.

**Architecture:** Start with a Swift Package named `PickoCore` because the repository has no app project yet and the product spec requires shared logic across iOS, iPadOS, and macOS. Keep Photos framework access out of the first package slice by modeling photo metadata with plain Swift types, so logic can be deterministic and unit-tested before platform UI work begins.

**Tech Stack:** Swift 5.9+, Swift Package Manager, XCTest.

---

## MVP Phases

### Phase 1: Shared Core Logic

Status: Completed and verified on 2026-05-31 with `swift test` passing 5 XCTest cases.

Create `Package.swift`, `Sources/PickoCore/`, and `Tests/PickoCoreTests/`.

Core responsibilities:
- `PhotoAsset`: local photo metadata and review status.
- `SimilarGroup`: group metadata, recommended keep IDs, keep count, and status.
- `ReviewSession`: session counters for reviewed, kept, pre-deleted, skipped, and estimated bytes.
- `DeletionQueue`: pre-delete basket with restore, clear, and estimated bytes.
- `ReviewStateStore`: in-memory state reducer for keep, pre-delete, skip, undo, and group selection.
- `SimilarityEngine`: deterministic metadata-first grouping by time window, location distance, media type, and thumbnail/perceptual hash.

### Phase 2: Photos Access Adapter

Add an Apple-platform target that wraps Photos framework authorization and asset metadata indexing. This should not run in Linux CI and should be separated from `PickoCore`.

### Phase 3: iOS MVP UI

Create a SwiftUI iOS app using native `TabView` for top-level navigation. Do not hand-roll a custom bottom tab bar; iOS 26 Liquid Glass should come from system components. Build onboarding, home, single review, similar group review, and pre-delete basket screens.

### Phase 4: macOS MVP UI

Create a SwiftUI macOS app with sidebar navigation, grid review, keyboard commands, multi-select, and a right-side inspector.

### Phase 5: Integration Verification

Connect platform UI to `PickoCore`, then add smoke UI tests for the main flows and performance checks for large libraries.

final class PickoCoreTests: XCTestCase {
    func testPreDeletedAssetsEnterDeletionQueueWithEstimatedBytes() {
        let asset = PhotoAsset(
            id: "a1",
            mediaType: .photo,
            creationDate: Date(timeIntervalSince1970: 10),
            location: nil,
            pixelWidth: 3000,
            pixelHeight: 2000,
            fileSizeBytes: 4_000_000,
            isFavorite: false,
            isEdited: false,
            isScreenshot: false,
            duration: nil,
            thumbnailHash: "aa",
            perceptualHash: "aa"

        XCTAssertEqual(store.asset(id: asset.id)?.status, .preDeleted)
        XCTAssertEqual(store.deletionQueue.itemIds, [asset.id])
        XCTAssertEqual(store.deletionQueue.estimatedBytes, 4_000_000)
    }

        let asset = PhotoAsset(id: "a1", mediaType: .photo, creationDate: Date(), location: nil, pixelWidth: 1, pixelHeight: 1, fileSizeBytes: 10, isFavorite: false, isEdited: false, isScreenshot: false, duration: nil, thumbnailHash: nil, perceptualHash: nil)

        store.apply(.preDelete(asset.id))
        store.undo()

        XCTAssertEqual(store.asset(id: asset.id)?.status, .unreviewed)
        XCTAssertTrue(store.deletionQueue.itemIds.isEmpty)
    }
}
